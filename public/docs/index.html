    Demographic API v1.1.0

    Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

    The Demographics API provides several unique value-added features for deriving actionable information from the publicly accessible Census Data. It enables market summaries, not only for defined geographic entities, like States, Counties and MSAs to list just a few, but also for custom markets defined by the user, such as radii or isochrone around any point on the map, or just any random enclosed region on the map.

    The Demographics API is organized around REST. Our API has predictable resource-oriented URLs, accepts well-defined request bodies mostly as JSON, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

    The Demographics API endpoints may broadly be categorized into the following modules:

    • User - Gaining Access to the API
    • Geo ID Census Block Search - Search for geographic identifiers associated with census blocks
    • Search Identifiers - Search for specific identifiers such as region name,ZIP codes
    • Address Translation - Translate addresses into standardized formats
    • Demographics Data - Provides access to demographics informations
    • Reverse Lookups - Allows users to input geographic coordinates (latitude and longitude) and obtain corresponding address information

    Authentication

    • HTTP Authentication, scheme: bearer .

    Every request sent to the Demographics API must be authenticated with an access token (JWT). You can obtain an access token when you log-in using the credentials provided to you. An access token is valid ONLY for 24 hours from the time it is generated.

    The obtained Access Token must be passed during all subsequent requests made to the API as standard Bearer tokens in the request header.

    GET /search/radius

    Returns a list of Census Block Geo IDs within radius

    Given a point in form of coordinates (longitude, latitude) and a scalar distance,it then draws a circle of the specified radius centered at the given point. The circle represents an area on the Earth's surface within which the demographics are to be searched. The endpoint returns all available Blocks contained within that circle.

    Parameters

    Name In Type Required Description
    long query number true Longitude of the given point
    lat query number true Latitude of the given point
    radius query number true Scalar distance/radius in miles
    geographicLevel query string false Geographic Level (default is Blocks) to show

    Example responses

    200 Response

    {
  "error": false,
  "geographicLevel": "Blocks",
  "count": 1,
  "data": [
    {
      "geoId": 120599602012055,
      "name": "Block 2055, Block Group 2, Census Tract 9602.01, Holmes County, Florida, USA"
    }
  ]
}

    GET /search/drivetime

    Returns a list of Census Block Geo IDs within drive time

    Given a point in form of coordinates (Longitude, Latitude) time in minutes and profile which provides directions for different modes of transportation, it then draws a polygonal shape. The shape represents an area on the Earth's surface which is reachable from the center via a drive within that time frame. The endpoint returns all available Blocks contained within that polygon.

    Parameters

    Name In Type Required Description
    long query number true Longitude of the given point
    lat query number true Latitude of the given point
    minutes query number true Drive time in minutes
    geographicLevel query string false Geographic Level (default is Blocks) to show
